# Professional Mindset Reflection Tool

SPA estática en `index.html` para reflexionar sobre decisiones profesionales sin lógica de correcto/incorrecto.

## Ver cambios localmente

```bash
python3 -m http.server 8000
```

Luego abre:

- http://localhost:8000

## Publicar en GitHub Pages

1. Sube este contenido al repositorio de GitHub (rama `main`).
2. En GitHub ve a **Settings → Pages**.
3. En **Build and deployment** selecciona:
   - **Source:** `Deploy from a branch`
   - **Branch:** `main`
   - **Folder:** `/ (root)`
4. Guarda y espera 1–3 minutos.

## URL correcta

Para repositorios de proyecto, la URL es:

- `https://<usuario>.github.io/<repositorio>/`

Ejemplo:

- Si el repo se llama `mentalidadtest`:
  - `https://vaneguerrero.github.io/mentalidadtest/`
- Si el repo se llama `metalidadtest` (sin **n**):
  - `https://vaneguerrero.github.io/metalidadtest/`

> Si ves 404, normalmente es porque GitHub Pages no está activado, el nombre del repo en la URL no coincide exactamente, o aún no terminó el deploy.
